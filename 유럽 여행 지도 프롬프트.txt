# 유럽 여행 지도 웹앱

## 프롬프트 원문
유럽 여행 다닐 때 나라별로 명소·역사·방문 팁을 확인할 수 있는 웹 지도를 만들어줘. 휴대폰과 노트북 모두에서 링크 하나로 바로 볼 수 있어야 하니, Claude Artifact(자기완결형 HTML, 외부 네트워크 요청 불가)로 만들어줘.

### 콘텐츠 요구사항
- 첫 번째 국가는 **포르투갈**, 완성도 100%로: 리스본·신트라·오비두스·나자레·코임브라·토마르·아베이루·포르투·브라가·기마랑이스·에보라·알가르브 12개 도시, 총 28개 명소.
- 명소마다 **역사 / 소개 / 방문 팁** 3개 항목을 한국어로 작성. 건립 연도·건축가·유네스코 지정 연도 등 사실관계는 웹 검색으로 검증해서 반영해줘 (추측 금지).
- 카테고리 분류: 성·요새 / 궁전 / 수도원·성당 / 자연·해안 / 거리·문화 5종. 검색창(이름 검색)과 카테고리 필터를 제공.

### 지도는 반드시 실제 지리 좌표 기반으로
일러스트로 대충 그린 국가 모양이 아니라 **진짜 국경선 데이터**를 써줘.
- 데이터 소스: Natural Earth (무료, 별도 인증 불필요)
  - 세계지도(국가 단위, 저해상도): `https://raw.githubusercontent.com/nvkelso/natural-earth-vector/master/geojson/ne_110m_admin_0_countries.geojson`
  - 국가 본토 윤곽 분리용(중해상도): `https://raw.githubusercontent.com/nvkelso/natural-earth-vector/master/geojson/ne_50m_admin_0_countries.geojson`
  - 국가 내 행정구역(고해상도, 지도에 진짜 지역 경계가 보이도록): `https://raw.githubusercontent.com/nvkelso/natural-earth-vector/master/geojson/ne_10m_admin_1_states_provinces.geojson`
- 작업 방식: Node.js로 위 GeoJSON을 다운로드 → 위경도를 SVG path로 직접 투영(project) → 결과를 정적 문자열로 HTML에 삽입. **런타임에 외부 API를 호출하지 않는다** (Artifact는 CSP로 외부 요청이 전부 막힘).
- 투영 방식:
  - 세계지도: 단순 등장방형(equirectangular). `x = (lon+180) * (960/360)`, `y = (90-lat) * (500/180)`, viewBox `0 0 960 500`.
  - 국가 상세지도: 위도 보정한 등장방형(로컬 투영) — 해당 국가 중심위도의 `cos(lat)`로 경도 폭을 보정한 뒤, viewBox에 맞춰 종횡비를 유지하며 스케일. 섬(예: 포르투갈의 아조레스·마데이라)은 본토와 멀리 떨어져 있어 bbox로 걸러내고 본토만 사용.
  - 도시/지점 좌표는 같은 투영 함수에 넣어 지도 위 정확한 위치에 점을 찍는다.
- 국가 상세지도는 국가 전체를 하나의 뭉뚱그린 도형으로 그리지 말고, admin-1(도/주/군) 경계까지 분리해서 그려줘 — 진짜 지도처럼 보이는 핵심 디테일.

### 디자인 레퍼런스 (세 사이트를 참고해서 통일된 스타일로)
- **https://www.domitur.pt/destinations/** → 전체 디자인의 기준. 웜그레이 배경, 가늘고 우아한 세리프 대제목, 지도 위 지점을 가리키는 **필(pill) 모양 라벨 + 얇은 직선 리더라인** 콜아웃. 필은 두꺼운 채움이 아니라 얇은 테두리의 담백한 스타일.
- **https://www.for-living.it/** → 지도 자체의 질감. 지역을 실제 경계선으로 세분화한 단색 지도, 마커는 점(core) + 그 주위 얇은 원형 링(ring)으로 표현(블러/네온 글로우 금지), 지도 배경에 아주 옅은 가로줄 텍스처.
- **https://www.aquamaremarine.com/** → 절제된 산세리프 타이포 톤. UI 라벨/본문에 깔끔한 그로테스크 계열 폰트, 버튼도 얇은 테두리 위주로 차분하게.
- **명시적으로 피할 것**: 알약 버튼에 그림자+두꺼운 채움+전부대문자 트래킹을 남발하는 "AI 생성 SaaS" 느낌, 확대/축소 버튼·범례 같은 툴 같은 UI chrome, 크림+테라코타의 흔한 AI 배색.

### 색상/타이포 시스템 (위 레퍼런스를 반영한 결과)
- 배경: 웜그레이 `#EDEAE1`(라이트) / `#15181A`(다크), 패널 `#F9F7F1` / `#1B1F21`
- 지도 기본 채움: 거의 흰색 `#FFFFFF` / `#20262A`, 경계선 `#C7BFA9` / `#3A4247`
- 브랜드·선택 강조색(내비-블루 계열): `#16233B` / `#6E93BE` — 활성 국가 채움, 활성 필터 필 채움에 사용
- 포인트 강조색(테라코타): `#B65C36` / `#E08A5B` — 도시 마커 점/링, 팁 카드 좌측 보더에만 사용 (내비색과 역할을 절대 섞지 않음)
- 카테고리별 태그색은 브랜드색과 별도로: 성·요새(스톤 브라운), 궁전(브론즈), 수도원·성당(슬레이트 블루), 자연·해안(틸), 거리·문화(와인)
- 폰트: 대제목은 얇은 세리프(`Iowan Old Style` 계열 스택), UI 라벨/네비는 깔끔한 지오메트릭 산세리프(`Century Gothic`/`Avenir Next` 계열 스택), 본문 장문(역사/소개/팁)은 가독성 좋은 휴머니스트 산세리프(`Seravek`/`Segoe UI`/`Malgun Gothic` 계열 스택, 한글 fallback 필수)
- 라이트/다크 테마 모두 지원: `prefers-color-scheme` + `:root[data-theme]` 양방향 오버라이드

### 화면 구조 (2단계)
1. **세계지도 화면(첫 화면)**: 중앙 상단에 브랜드 필, 큰 세리프 헤드라인, 아래 세계지도. 국가는 기본 흰색, 콘텐츠가 준비된 국가만 브랜드색으로 채움 + 필 라벨과 리더라인으로 콜아웃. 필/국가 도형을 클릭하면 국가 상세화면으로 전환.
2. **국가 상세화면**: 뒤로가기 텍스트 링크, 국가명(영문 세리프 대제목 + 한글 보조), 통계 한 줄(도시 수·명소 수·완성도). 좌측(또는 위) — 실제 행정구역 경계가 보이는 지도에 도시별 점+링 마커, 클릭 시 해당 도시 카드 섹션으로 스무스 스크롤. 우측(또는 아래) — 명소 검색창(밑줄 스타일 input), 카테고리 필터(얇은 테두리 필, 활성 시 브랜드색 채움), 도시 바로가기 텍스트 링크 목록. 아래로 도시별 섹션 + 명소 카드 그리드(`<details>` 아코디언, 요약 상태에서 이름/카테고리/한줄 훅 표시, 펼치면 역사·소개·팁 노출).

### 구현/검증 체크리스트
- 전체 데이터(세계 지도 path, 국가 행정구역 path, 도시 좌표, 명소 콘텐츠 JS 배열)는 빌드 스크립트로 하나의 정적 HTML 파일에 합쳐 넣는다 (템플릿 + 플레이스홀더 치환 방식 추천 — 대용량 path 문자열을 직접 타이핑하지 말 것).
- `node --check`로 삽입된 `<script>` 블록 문법 검증.
- 실제 브라우저(가능하면 Claude in Chrome 등 브라우저 자동화 도구)로 열어서: 세계지도 로딩 → 국가 클릭 → 상세지도 렌더링 → 도시 마커 클릭 스크롤 → 검색/필터 동작 → 카드 펼치기까지 직접 확인.

### 확장 방법 (다음 국가 추가 시)
- 같은 파이프라인으로 새 국가의 admin-0(본토 분리)·admin-1(행정구역)·주요 도시 좌표를 뽑아 데이터만 추가하면 됨.
- 세계지도의 `country-path.active` 클래스와 콜아웃 필을 국가별로 데이터 배열화해서 여러 국가를 순서대로 켜지도록 리팩터링.
- 국가 선택 순서는 사용자가 별도로 요청한 "포르투갈 거주 기준 하반기(8월 말~2월) 여행하기 좋은 나라 순서"를 반영.
